# n8n-nodes-google-news-decoder

## Overview

## How Google News URL Decoding Works

Google News RSS feeds use encoded URLs instead of direct article links. There are two formats:

### Old Format (direct base64 decode)
The base64 payload contains a protobuf structure with the original URL embedded as a UTF-8 string in field 2. Decoding is instant (no network calls).

### New Format (API-based decode) — Most Common Now
The base64 payload contains an encrypted/obfuscated article ID. Decoding requires two HTTP requests:

1. **GET** `https://news.google.com/articles/{base64}` — fetch article page HTML
2. Extract `data-n-a-sg` (signature) and `data-n-a-ts` (timestamp) from the HTML
3. **POST** `https://news.google.com/_/DotsSplashUi/data/batchexecute` with payload:
   - RPC name: `Fbv4je`
   - Request type: `garturlreq`
   - Includes the base64 article ID, timestamp, and signature
4. Parse response: split by `\n\n`, take second section, `JSON.parse()[0][2]` → `JSON.parse()[1]` = decoded URL

## Key Files Detail

### `nodes/GoogleNewsDecode/utils/decoder.ts`
Core decoding logic with dependency injection for HTTP requests:

- `extractBase64FromUrl(url)` — Extracts base64 from `/rss/articles/`, `/articles/`, or `/read/` paths. Only accepts `news.google.com` hostname.
- `tryDirectDecode(base64)` — Attempts old-format decode by looking for `http` in decoded bytes.
- `decodeViaApi(base64, httpRequest)` — New-format decode using batchexecute API (2-step).
- `decodeGoogleNewsUrl(url, httpRequest)` — Main entry point. Tries direct decode first, falls back to API.
- `HttpRequestFn` type — Abstraction over HTTP client (allows n8n's `this.helpers.httpRequest`).
- `DecodeResult` interface — `{ status, decodedUrl?, method?, message? }`

### `nodes/GoogleNewsDecode/GoogleNewsDecode.node.ts`
n8n node implementation:

- **Parameters**:
  - `urlField` (string, default: `"link"`) — Input field name containing the Google News URL, OR a direct URL starting with `https://`
  - `outputField` (string, default: `"decodedUrl"`) — Output field for decoded URL
  - `delay` (number, default: `0`) — Delay in ms between requests for rate limiting
- **Batch processing**: Iterates `this.getInputData()` items, processes each sequentially
- **Output fields added**: `[outputField]` (decoded URL), `_decodeMethod` (`"base64"` or `"api"`), `_decodeError` (on failure)
- **Error handling**: Supports `this.continueOnFail()` for graceful error handling
- **URL detection**: If `urlField` value starts with `http://` or `https://`, uses it as the URL directly instead of treating it as a field name lookup

### `n8n-workflow.json`
Complete n8n workflow: Google News RSS → Decode URLs → AI Classify → Telegram

**Workflow flow**:
```
Schedule Trigger (hourly)
  → Fetch RSS Feed (Google News Vietnamese topics)
  → XML (parse RSS XML)
  → ExtractClusters (Code: extract headlines, sources, related articles)
  → Split Batches (Code: split into batches of 8 clusters)
  → Loop Over Items (SplitInBatches node)
    ├── [done] → Merge All Batches → PrepareInput → AI Classify Topics (GPT-4o) → BuildTelegramMessage → Send Telegram
    └── [loop] → Flatten URLs → Google News Decode → Rebuild Clusters → Loop Over Items
```

**Key nodes in the decode loop**:
- **Flatten URLs** (Code): Extracts all Google News URLs from batch clusters into individual items. Each item has `{ link, _clusterIndex, _type, _relatedIndex }` for mapping back. The batch metadata is stored in `_batch` field of first item.
- **Google News Decode**: Community node (`@19americano/n8n-nodes-google-news-decoder`). Reads `link` field, outputs `decodedUrl`, 500ms delay.
- **Rebuild Clusters** (Code): Collects decoded URLs from all items, maps them back into cluster structure as `mainLinkResolved` and `resolvedLink` fields.

## Known Issues & Gotchas

1. **`NodeConnectionType` is type-only in n8n-workflow** — Cannot use `[NodeConnectionType.Main]` for inputs/outputs. Use `['main'] as any` instead.
2. **copyfiles glob on Windows** — `"nodes/**/*.svg"` glob doesn't work reliably. Use explicit file paths in the build script.
3. **Google API rate limiting** — When processing many URLs, use the `delay` parameter (500-1000ms recommended).
4. **batchexecute response format** — Response is double-nested array `[["wrb.fr",...],["di",...]]`, NOT triple-nested. Parse: `response.split('\n\n')[1]` → `JSON.parse()[0][2]` → `JSON.parse()[1]`.
5. **URL Field parameter** — Accepts both a field name (e.g., `link`) and a direct URL (e.g., `https://news.google.com/rss/articles/...`). Auto-detected by checking if value starts with `http://` or `https://`.
